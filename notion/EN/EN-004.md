## 1. Objetivo y alcance

Dejar listo el camino para que **API + frontend** corran en un entorno accesible y **reproducible
desde los contenedores**, de forma que la demo no dependa de la máquina de nadie.

> **Nota de honestidad.** Según `docs/SPC_Entrega_Despliegue_Valentin.md`, **aún no se ha
> desplegado nada en vivo** (la Fase 4 sigue pendiente). Lo que está **hecho** es toda la
> **infraestructura de despliegue**: con un clic en Render el servicio se construye solo. Abajo se
> documenta lo preparado y lo que falta para tener la URL pública.

## 2. Qué se hizo

### 2.1. Blueprint de Render para la API (`render.yaml`)

- Servicio **web** `spc-api`, runtime **docker**, construido desde `./Dockerfile`, plan `free`.
- **`healthCheckPath: /health`** → Render verifica la salud antes de enrutar tráfico.
- **`autoDeploy: true`** → cada push a la rama reconstruye y redepliega.
- **CORS por variable de entorno** (`SPC_CORS_ORIGINS`): en producción se cambia `"*"` por el
  origen real del frontend.

### 2.2. La API es "deploy-ready"

- Endpoint **`/health`** (`src/spc/api/main.py:214`) para el *health check* de la plataforma.
- **Bind a `0.0.0.0:$PORT`** (lo inyecta la plataforma) — ver **EN-003**.
- **CORS configurable** por entorno; artefactos **horneados** en la imagen (arranca sin GPU ni datos crudos).

### 2.3. Frontend desacoplado del backend por configuración

- El cliente HTTP resuelve la base de la API por **`VITE_API_BASE_URL`** (`frontend/src/api/client.ts`):
  el mismo build apunta a `localhost` en desarrollo o a la URL pública de la API en producción.
- **Build estático** con Vite (`npm run build` → `frontend/dist/`): se publica como *static site*
  (Render Static / Vercel / Netlify) sin servidor propio.

## 4. Resultados (criterio de "hecho")

- ✅ **Despliegue reproducible desde los contenedores**: `render.yaml` + `Dockerfile` construyen la
  API sin pasos manuales.
- ✅ Health check, CORS por entorno y bind a `$PORT` listos para una plataforma PaaS.
- ✅ Frontend con build estático y **URL de API por variable de entorno** (desacoplado del backend).
- 🟡 **URL accesible de la solución:** pendiente de publicar (API en Render + frontend estático +
  `SPC_CORS_ORIGINS` apuntando al origen real del front). Es el paso final de Fase 4.

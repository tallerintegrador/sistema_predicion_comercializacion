# EN-001 — Configurar repositorio y estructura del proyecto

> **Habilitador técnico.** No entrega valor de usuario directo: establece el **esqueleto del
> repositorio, el empaquetado, el tooling de calidad y las convenciones** sobre los que se construyen
> todas las HU.

- **Ámbito:** Infraestructura / tooling. **Transversal** (precede a Fase 1–3).
- **Archivos clave:** `pyproject.toml`, `.gitignore`, `requirements.txt`, `requirements-api.txt`,
  `README.md`, layout `src/spc/`.

---

## 1. Objetivo y alcance

Dejar el repositorio listo para trabajar de forma **reproducible, testeable y mantenible**:

- repositorio Git con modelo de ramas y convención de commits,
- paquete Python instalable con layout `src/`,
- dependencias **pineadas exactamente** (reproducibilidad de extremo a extremo),
- tooling de calidad (lint, formato, tipos, tests) configurado en un solo lugar,
- estructura de carpetas por **capas** que sostiene la arquitectura del sistema.

Punto de partida: el proyecto venía de un único `eda.py` de ~1700 líneas. Este habilitador lo
convierte en el **paquete modular `spc`**, testeado y con tooling.

## 2. Qué se hizo

### 2.1. Repositorio Git y modelo de ramas

- Repositorio Git con remoto en **GitHub**. Autor: *Valentin Fernandez*.
- **Ramas:** `develop` (integración del día a día) y `main` (estable / base de PRs).
- **Convención de commits:** español, estilo *Conventional Commits* (`feat:`, `fix:`, `feature:`…).
- **`.gitignore`** excluye lo pesado/generado: `venv/`, cachés (`.pytest_cache`, `.mypy_cache`,
  `.ruff_cache`), `data/raw/*` (datos crudos, no se versionan — solo `.gitkeep`),
  `data/processed/`, `figures/`, `data/synthetic/*`, y basura de IDE/SO.

### 2.2. Empaquetado: layout `src/` instalable (`pyproject.toml`)

- Paquete **`spc`** v0.1.0, `requires-python >= 3.11`, build con `setuptools`
  (`package-dir = {"" = "src"}`).
- **Layout `src/`** deliberado: los tests importan `from spc... import ...` **sin trucos de ruta** y
  el paquete es instalable (no depende del CWD).
- Instalación reproducible con entorno virtual + editable:

  ```powershell
  python -m venv venv
  venv\Scripts\python -m pip install -e .[dev]
  ```

- **Entry points** declarados (`[project.scripts]`): `spc-eda`, `spc-models`,
  `spc-train-regresion`, `spc-train-clasificacion`.

### 2.3. Dependencias pineadas + separación runtime / desarrollo

- **Versiones exactas** en `pyproject.toml` (p. ej. `pandas==3.0.3`, `numpy==2.4.6`,
  `scikit-learn==1.9.0`, `lightgbm==4.6.0`, `xgboost==3.2.0`, `fastapi==0.137.1`,
  `pydantic==2.13.4`). Razón crítica: con otras versiones de numpy/pandas/scikit-learn los `.joblib`
  **pueden no cargar**.
- **Extra `[dev]`**: `pytest`, `ruff`, `black`, `mypy`, `pandas-stubs`, `jupyter`, `ipykernel`.
- **Dos archivos de requisitos** por propósito:
  - `requirements.txt` — entorno completo (EDA + entrenamiento + API).
  - `requirements-api.txt` — **subconjunto de runtime** con pines exactos para la imagen de servicio
    (deben coincidir con los de `pyproject.toml`).

### 2.4. Tooling de calidad (todo en `pyproject.toml`)

| Herramienta | Config | Propósito |
|-------------|--------|-----------|
| **ruff** | `line-length=100`, `select = E,F,I,UP,B,C4,SIM`, `ignore=E501` | lint + orden de imports |
| **black** | `line-length=100`, `py311` | formato |
| **mypy** | `packages=["spc"]`, `mypy_path=src`, `no_implicit_optional`, `warn_unused_ignores` | tipos |
| **pytest** | `testpaths=["tests"]`, `pythonpath=["src","tests"]`, `filterwarnings` | tests |

Comando de calidad:

```powershell
venv\Scripts\python -m pytest        # tests
venv\Scripts\ruff check src tests    # lint
venv\Scripts\black --check src tests # formato
venv\Scripts\mypy src                # tipos
```

### 2.5. Estructura de carpetas por capas

```
sistema_prediccion_comercializacion/
├─ src/spc/              # paquete principal
│  ├─ api/               # FastAPI: main, routers/, schemas/, dependencies, errors
│  ├─ service/           # capa de negocio: adaptador + *_service + artefactos
│  ├─ models/            # motor ML: regresion, clasificacion, clustering, runner
│  ├─ features/          # ingeniería de features (temporales, perfiles)
│  ├─ data/              # carga/integración de fuentes (loaders, schemas, holidays)
│  ├─ eda/               # pipeline de EDA (analysis, quality, viz, reporting)
│  ├─ config/            # Settings (rutas, semilla 42, estilo de figuras)
│  ├─ synthetic/         # generación de datos sintéticos de apoyo
│  └─ utils/             # logging, métricas, formatters, serialización
├─ scripts/              # entry points CLI (run_eda, train_*, generate_notebook)
├─ tests/                # pytest sobre datos sintéticos (+ tests/api)
├─ docs/                 # reportes + decisiones/ (ADRs 0002–0007) + contrato_datos
├─ data/                 # raw (gitignored) · processed · synthetic
├─ models/               # artefactos .joblib + .meta.json (versionados)
├─ figures/              # figuras del EDA (gitignored)
├─ notebooks/            # notebook reproducible del EDA
└─ notion/               # esta documentación (EN/ y HU/)
```

**Dependencias hacia adentro:** `api → service → motor`. El motor **no importa nada de la API**; el
servicio **no importa `spc.api`**. Esto sostiene la separación de capas en todo el sistema.

### 2.6. Reproducibilidad como principio

- **Semilla fija = 42** en `config/Settings` (entrenamientos deterministas).
- Artefactos **portables**: entrenan en GPU, predicen en CPU; se sirven sin GPU ni `data/raw`.
- La suite de tests corre **sin datos crudos ni GPU** (entrena artefactos diminutos sintéticos).

## 3. Resultados (criterio de "hecho")

- ✅ `pip install -e .[dev]` deja el paquete `spc` instalado e importable.
- ✅ `pytest`, `ruff`, `black --check`, `mypy` ejecutan desde un solo origen de config.
- ✅ Layout `src/` + `pythonpath` permiten `from spc... import ...` en tests sin hacks de ruta.
- ✅ Dependencias pineadas → mismas versiones reproducen los `.joblib`.
- ✅ Estructura por capas con dependencias hacia adentro (motor sin conocer API).
- ✅ `.gitignore` evita versionar datos pesados/artefactos generados.

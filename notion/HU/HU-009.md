# HU-009 — Visualización de resultados

> **Como** usuario de la demo, **quiero** ver gráficos de los resultados (pronóstico y riesgo),
> **para** interpretar las predicciones.

- **Capa:** Frontend web (React + recharts). **Sprint 2: Semana 12.**
- **Estado:** ✅ Implementada. Gráficos por dominio + tablas filtrables.
- **Código clave:** `frontend/src/components/charts/` (`SalesChart`, `PurchasesChart`,
  `InventoryRisk`, `SerieChart`), `components/result/ResultFilters.tsx`, `components/ResultTable.tsx`.
- **Decisiones:** ADR-0012 (frontend web), 0017 (identidad visual / tokens), 0021 (filtros de resultado).

---

## 1. Objetivo y alcance

Convertir la respuesta JSON de la API en **visualizaciones legibles**: que el usuario **vea** el
pronóstico de demanda, la recomendación de compra y el **riesgo/segmento** de almacén, no que lea un
JSON. Se construye sobre **HU-008** (que trae el resultado) y usa **recharts** + el sistema de
diseño por tokens (ADR-0017).

## 2. Qué se hizo

### 2.1. Un gráfico por dominio (`components/charts/`)

- **`SalesChart.tsx`** — serie temporal del **pronóstico de demanda** (`forecast`) de `/sales`.
- **`PurchasesChart.tsx`** — visual de la **recomendación de reposición** (cantidad / punto de
  reorden) de `/purchases`.
- **`InventoryRisk.tsx`** — **riesgo de quiebre** y segmento de la tienda de `/inventory`
  (clase + nivel de riesgo).
- **`SerieChart.tsx`** — componente de serie reutilizable que comparten las vistas.

### 2.2. Tablas + filtros para explorar el resultado

- **`ResultTable.tsx`** muestra el detalle por `(date, store_id, product_id)`.
- **`result/ResultFilters.tsx`** + el hook **`hooks/useResultFilters.ts`** permiten filtrar por
  tienda/producto/dimensión (ADR-0021), de modo que un resultado grande sea navegable.

### 2.3. Coherencia visual (ADR-0017)

- Los gráficos consumen los **tokens de diseño** (marca índigo, acento teal, semánticos) definidos en
  `frontend/src/index.css`, no colores sueltos → la visualización es consistente con el resto de la app.

## 4. Resultados (criterio de "hecho")

- ✅ **Al menos un gráfico del pronóstico** (serie temporal de `forecast` en `SalesChart`).
- ✅ **Visualización del riesgo / segmento** (`InventoryRisk`) y de la reposición (`PurchasesChart`).
- ✅ Resultado **explorable**: tabla por `(fecha, tienda, producto)` con filtros.
- ✅ Estilo coherente con el sistema de diseño (ADR-0017), sin colores hard-codeados.

## 5. Cómo probar

```powershell
cd frontend; npm run dev
# Login → cargar la demo o un Excel → Ejecutar → ver el gráfico y la tabla del dominio
```

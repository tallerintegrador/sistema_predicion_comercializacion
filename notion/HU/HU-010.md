# HU-010 — Demo funcional de extremo a extremo

> **Como** equipo, **queremos** una demo que recorra el flujo completo (datos → predicción →
> visualización), **para** presentar el proyecto.

- **Capa:** Integración frontend + API + motor. **Sprint 2: Semana 12.**
- **Estado:** ✅ Funciona de extremo a extremo **en local**; despliegue en vivo es **EN-004**.
- **Código clave:** `frontend/src/demo/retailDemo.ts` (datos de demo), `frontend/src/App.tsx`
  (ruteo), API `src/spc/api/` (tres dominios + auto), motor `src/spc/models/` (artefactos congelados).
- **Apoyo:** guion de demo (Fase 3.7) y `docs/guia_probar_api.md`.

---

## 1. Objetivo y alcance

Tener un **recorrido completo y repetible** que muestre el producto sin fricción: el usuario entra,
**trae datos** (o usa la demo cargada), **predice** los tres dominios y **ve los resultados** en
gráficos. Cubre la cadena **datos → contrato → servicio → motor → respuesta → visualización**.

## 2. Qué se hizo

### 2.1. Datos de demo listos para usar (`demo/retailDemo.ts`)

- Mini cadena minorista **3 tiendas × 60 días** con señal **semanal + promociones + precio/clima** y
  **columnas libres** (`unidades_vendidas`, `ingresos`), pensada para alimentar **Ventas** y
  **Compras** sin que el usuario traiga nada.
- Demuestra el **motor agnóstico auto-entrenado** (ADR-0023): el cliente declara su esquema y trae
  columnas arbitrarias; el backend entrena el ganador al vuelo y responde.

### 2.2. El flujo completo conectado

- **Ruteo** del frontend (`App.tsx` + secciones por permiso) lleva por Catálogo → Ventas → Compras →
  Almacén.
- Cada paso **consume la API real** (HU-005/006/007 y los `/auto/*`), no datos simulados en el front.
- El resultado se **visualiza** con los gráficos de **HU-009**.

### 2.3. Cubre los tres dominios

- **Ventas** (pronóstico), **Compras** (reposición) y **Almacén** (riesgo/segmento) corren sobre el
  **mismo histórico** reutilizado (contrato compartido, TA-001), igual que en producción.

```powershell

cd frontend; npm run dev                          # frontend

# 2) Cargar "demo" o subir Excel

# 4) Compras → ver recomendación
# 5) Almacén → ver riesgo/segmento
```

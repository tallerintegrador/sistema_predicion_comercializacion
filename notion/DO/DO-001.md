# DO-001 — Documentación de la API

> **Tarea de documentación.** Documentar los endpoints de la API (parámetros, respuestas y
> ejemplos), preferentemente con un estándar tipo OpenAPI/Swagger.

- **Ámbito:** Documentación / API. **Sprint 2: Semana 12.**
- **Estado:** ✅ Cubierta. Swagger/OpenAPI autogenerado + guía paso a paso + ejemplos por dominio.
- **Fuentes:** `src/spc/api/main.py` (OpenAPI), `docs/guia_probar_api.md`, `docs/contrato_datos.md`,
  `examples/api/*.json`.

---

## 1. Objetivo y alcance

Que cualquiera pueda **entender y probar la API** —parámetros, respuestas y errores— sin leer el
código: documentación **interactiva** (Swagger) generada por el propio contrato, más una guía para
no-programadores y peticiones de ejemplo reales por dominio.

## 2. Qué se hizo

### 2.1. Swagger / OpenAPI autogenerado (`src/spc/api/main.py`)

- FastAPI publica **`/docs`** (Swagger UI) y **`/openapi.json`** a partir de los esquemas Pydantic.
- App con **título y descripción** (`main.py:161`) y **tags por dominio** (`openapi_tags`), de modo
  que SALES / PURCHASES / INVENTORY aparecen agrupados y navegables.
- Cada esquema lleva un **ejemplo** que se ve en Swagger → la documentación de petición/respuesta
  **la genera el contrato**, no se escribe a mano (no se desincroniza del código).
- **Errores documentados** (400/422) con el **sobre de error uniforme** `{error:{type,message,details}}`.

### 2.2. Guía para probar sin programar (`docs/guia_probar_api.md`)

- Paso a paso: levantar el servidor, abrir Swagger y enviar las peticiones de ejemplo con clics.
  Pensada para verificar el servicio **sin internet ni datos crudos**.

### 2.3. Contrato como fuente de verdad (`docs/contrato_datos.md`)

- Define los campos genéricos, los tres dominios y los ejemplos de entrada/salida; es el documento
  que la API implementa (ver **TA-001**).

### 2.4. Peticiones de ejemplo versionadas (`examples/api/`)

- `ventas_request.json`, `compras_request.json`, `almacen_request.json` y los `auto_*` (ADR-0023):
  cuerpos listos para *Try it out* en Swagger o para `Invoke-RestMethod`.

## 4. Resultados (criterio de "hecho")

- ✅ **Cubre todos los endpoints**: los tres dominios (`/sales`, `/purchases`, `/inventory`), los
  `/auto/*`, `/catalog`, `/health` y los de entrenamiento aparecen en Swagger.
- ✅ **Incluye ejemplos de petición y respuesta** (en los esquemas y en `examples/api/`).
- ✅ Estándar **OpenAPI** (autogenerado) → la doc no se desincroniza del código.
- ✅ Guía complementaria para usuarios no técnicos.

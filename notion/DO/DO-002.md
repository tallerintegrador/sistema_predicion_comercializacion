# DO-002 — Documentación viva en docs/

> **Tarea de documentación.** Mantener la documentación viva del proyecto (Plan Maestro, decisiones
> y guía de uso) versionada en `docs/`.

- **Ámbito:** Documentación / proceso. **Sprint 2: Semana 12.**
- **Estado:** ✅ Cubierta. `README` de uso + `docs/` con plan maestro, ADRs y reportes versionados.
- **Fuentes:** `README.md` (raíz), `docs/plan_maestro_spc.md`, `docs/decisiones/` (ADR 0002–0023),
  `docs/` (reportes y guías), `notion/` (esta documentación del backlog).

---

## 1. Objetivo y alcance

Que las **decisiones y el estado del proyecto** vivan en el repositorio, versionados junto al código,
y no en la cabeza de una persona: un **README** de instalación/uso, un **Plan Maestro**, un registro
de **decisiones (ADR)** y reportes por fase, todos actualizados a medida que el proyecto avanza.

## 2. Qué se hizo

### 2.1. README con instrucciones de instalación y uso

- `README.md` (raíz) explica cómo instalar (`pip install -e .[dev]`), correr la calidad
  (pytest/ruff/black/mypy) y levantar la API; `frontend/README.md` documenta la app web.

### 2.2. Plan Maestro y reportes por fase (`docs/`)

- `docs/plan_maestro_spc.md` (visión y fases), más reportes vivos: `reporte_eda.md`,
  `reporte_regresion_2a.md`, `reporte_clasificacion_2b.md`, `reporte_clustering_2c.md`, las guías
  (`guia_probar_api.md`) y la documentación de Fase 3 en `docs/fase-3/`.

### 2.3. Registro de decisiones de arquitectura (`docs/decisiones/`)

- **ADR 0002–0023** documentan cada decisión de fondo con su contexto: modelo de regresión, capa de
  API (0007), modos de ejecución (0008), modelo congelado (0009), persistencia del corpus (0011),
  frontend (0012), control de acceso (0014), identidad visual (0017), predicción agnóstica (0023)…
- El formato ADR deja **trazable el porqué** de cada decisión, no solo el qué.

### 2.4. Documentación del backlog para Notion (`notion/`)

- Esta carpeta (`notion/EN|RN|TA|HU|DO/`) traduce el trabajo a reportes por ítem del backlog, con
  código puntual, resultados y capturas sugeridas — fuente directa del reporte en Notion.

### 2.5. Documento de entrega/alineación

- `docs/SPC_Entrega_Despliegue_Valentin.md` (estado Fase 3 → Fase 4) y
  `docs/alineacion_frontend_backend.md` (conciliación UI ↔ backend) mantienen al equipo sincronizado.
